# GS-Customized-Email-Merge

🚀 **強大、靈活且支援動態欄位對應的 Gmail 郵件合併工具，專為解決大量產學合作信件與連結失效問題而設計。**

本工具特別針對 Gmail 編輯器對連結標籤的截斷問題進行了優化，並捨棄了固定欄位限制，改採「**自動標頭偵測 (Header-based Mapping)**」，讓使用者能自由安排試算表格式。

## 🌟 核心功能

* **動態欄位偵測**：程式會自動讀取試算表第一列（Header），只要標頭符合 `{{標籤}}` 格式，即可自動進行資料套印。
* **標籤位置自由化**：不再限制固定欄位。只要在試算表中定義 `{{email}}` 與 `{{result}}`，其餘資訊如姓名、連結等可任意擺放。
* **英文標籤優化**：全面支援英文標籤（如 `{{link}}`），避免中文標籤在 Gmail 連結視窗中被截斷或產生亂碼。
* **智慧連結處理**：
* **文字帶超連結**：支援在 Gmail 撰寫草稿時選取文字並插入 `{{link}}` 標籤。即使 Gmail 自動補上 `http://` 前綴，系統也能自動修復並替換為正確網址。
* **純連結顯示**：使用 `{{rawlink}}` 標籤可直接在郵件中印出完整長網址。


* **Emoji 完美支援**：自動將 Emoji 轉換為 HTML 安全代碼，確保跨平台顯示正常。
* **視覺化操作與預覽**：支援從 Gmail 草稿匣選擇範本，並在正式寄出前預覽第一封郵件的套印效果。

## ⚠️ 重要聲明與使用風險 (Disclaimer & Usage Risks)

在使用本工具前，請務必閱讀並理解以下說明。使用者須對其發送行為負全部責任：

1. **Google 寄送配額限制**：一般 Gmail 帳號每日發送上限為 **500 封**，Workspace 帳號為 **2000 封**。
2. **垃圾郵件標註風險**：若內容被頻繁標註為垃圾郵件，將影響網域聲譽並可能導致帳號停權。
3. **法律合規性**：請確保您的發送行為符合當地法律（如《濫發商業電子郵件管理條例》）。

## 🛠️ 試算表設定 (Spreadsheet Setup)

本系統會自動偵測第一列（Row 1）的標頭名稱。請確保包含以下必要欄位：

| 標籤 (Header) | 說明 | 狀態 |
| --- | --- | --- |
| `{{email}}` | 收件者的電子郵件地址。 (亦支援直接寫 `Email`) | **必要** |
| `{{result}}` | 系統回填「已寄出」或「已建草稿」紀錄的欄位。 | **必要** |
| `{{link}}` | 放置目標跳轉網址（如：`https://example.edu.tw`）。 | 建議 |
| `{{name}}` | 收件人姓名或稱呼。 | 建議 |
| `{{company}}` | 公司或單位名稱。 | 選用 |
| `{{任何英文名}}` | 您可自訂任何標籤（如 `{{project}}`），程式會自動抓取對應。 | 選用 |

## 📋 變數與連結操作說明

### 1. 一般文字替換

在 Gmail 草稿中直接輸入對應標籤。例如：`親愛的 {{name}} 您好，感謝來自 {{company}} 的支持。`

### 2. 文字帶超連結 (推薦方式)

* **操作**：選取文字（例如：**點我回覆表單**），按下 `Ctrl + K`。
* **網址欄位**：輸入 **`{{link}}`**。
* **系統修正**：Gmail 可能自動變為 `http://{{link}}`，系統會自動辨識並還原為您試算表中填寫的完整連結。

### 3. 直接顯示網址

在草稿中使用 `{{rawlink}}` 標籤，郵件中會直接印出該欄位的完整字串。
